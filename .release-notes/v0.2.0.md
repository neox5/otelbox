# Release v0.2.0

Major feature release adding YAML configuration, OTEL push support, and container deployment.

## Features

**Configuration System:**

- YAML-based configuration with four sections: simulation, metrics, export, settings
- Multi-clock support with independent intervals
- Value cloning and reset-on-read behavior for coherent metric representations
- Protocol-specific metric naming (Prometheus vs OTEL)
- Seed configuration for reproducible simulations

**OTEL Integration:**

- Push-based metric export via OTLP protocol
- Support for both gRPC (default, port 4317) and HTTP (port 4318) transports
- Configurable resource attributes and custom headers
- Flexible push and read intervals

**Container Support:**

- Multi-platform images for linux/amd64 and linux/arm64
- Available via `ghcr.io/neox5/obsbox:latest`
- Default configuration included in image

**Documentation:**

- Complete configuration guide with examples and patterns
- Detailed reference documentation for all parameters

## Dependencies

- simv v0.5.0 - explicit seed initialization required
- OpenTelemetry SDK for OTLP export
- Prometheus client library

## Installation

### Container

```bash
podman pull ghcr.io/neox5/obsbox:v0.2.0
```

### Binaries

Download from [GitHub Releases](https://github.com/neox5/obsbox/releases/tag/v0.2.0)

### Source

```bash
go install github.com/neox5/obsbox/cmd/obsbox@v0.2.0
```
